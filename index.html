Simple Programming for PocketMine (Minecraft Bedrock Edition) Servers

MadCommands makes you able to override existing commands or define your own commands using a simple BASIC-like programming language. You can even let other players write programs for your server without giving them permission to manage the whole server.

MadCommands in action

What's new in MadCommands 2.5 

Feature: Added json and posturl functions to simplify integrations with external RESTful APIs
Feature: Added playereffects function returning player's currently active effects
Change: playerstatus now includes player's Xbox Live ID, speed and motion vector
Change: playerstatus now includes flying, sleeping, sprinting, sneaking, and onfire flags
Change: Player's level, position, and Xbox Live ID are now returned by playerstatus even for offline players
Change: The playerinventory can now optionally include empty slots in the result
Change: The in and notin operators can now be used to search for multiple values in a string in one command
Change: Added xuid (Xbox Live user ID) to the list of values returned by the playerstatus function for online players
Fix: playerstatus function now works for offline players as well, info returned is limited to "online" and "op" flags
Fix: boss command updated to work with current version of PocketMine again
Fix: areainfo funcion no longer failing when trying to get specific whitelisted player's area flags
Fix: cmd help no longer throwing exception under PHP 8
Fix: help for the xp command no longer incorrectly mentions the move command instead
Fix: Typo in help for the "async" command
Fix: Exception thrown from external command executed with "asop" option should no longer prevent the op permission from being reset
