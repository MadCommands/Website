install

--------

 

The /cmd install command can be used to install pre-defined command procedures from an external repository.

 

Typing simply /cmd install with no extra parameters will list all available commands in the repository.

 

Use /cmd install <command_name> to install specific command of your choice.

 

By default, madhon's repository is used. You can repoint the plugin to a different repository by modifying the install-source config option.

 

EXAMPLES:

/cmd install

/cmd install countdown

create

-------

 

Use /cmd create <command_name> to create a new command procedure.

 

If you want your procedure to be registered as a regular command, you can also provide optional description and usage for the standard /help command:

 

/cmd create <command_name> <description> | <usage>

 

After the command is registered, users can use /help <command_name> to check the command's description and usage.

 

Whether new commands are automatically enabled and/or registered when created can be controlled by modifying the enable-new-commands and register-new-commands config options.

 

EXAMPLES:

/cmd create hello

/cmd create hello This command says hello | /hello

add

----

 

Use /cmd add to add a new command line to the end of your custom command procedure:

 

/cmd add <command_name> [asop|ascon] <command> [<parameter>] ...

 

Note that regular game commands need to be typed exactly as you would type them in the chat - including the leading forward slash character.

 

If you precede your command with ascon, this line will be executed from the console context. This has the advantage of bypassing any permission checking, but can only be used for commands that do work from the console context.

 

If you precede your command with asop, the player executing your procedure will be temporarily given Op permissions for the time of this line's execution, but the command itself will still be executed from the player's context.

 

EXAMPLES:

/cmd add makememember /me is now member

/cmd add makememember ascon /setgroup %p% Member

/cmd add makememember message %p% You are now member

edit

-----

 

You can use the /cmd edit command to replace an existing line in your custom command procedure, or to create one with any line number of your choice:

 

/cmd edit <command_name> <line_number> [asop|ascon] <command> [<parameter>] ...

 

If a line with the specified number already exists, it will be replaced with your new command. If the line does not exist, it will be added without modifying any other lines of the procedure.

 

Using a custom line number is very helpful when you want to jump to another part of your procedure using the goto command but don't know how many lines of code you will need to write between the two parts yet. Common practice used by BASIC programmers is to initially number all lines by the multiplies of 10, so that you can insert extra commands in between without having to renumber the following lines.

 

Type /cmd help add to learn about the ascon and asop options.

 

EXAMPLES:

/cmd edit hello 10 /say Nice to see you here

/cmd edit hello 5 /say Hello everyone!

/cmd edit hello 7 exit

 

------

insert

-------

 

Use /cmd insert when you need to insert an extra command line between already existing lines of your procedure and you have not skipped any line number for later use:

 

/cmd insert <command_name> <line_number> [asop|ascon] <command> [<parameter>] ...

 

This command will increment all line numbers immediately following the line number you're inserting, until a skipped line number of end of procedure is found.

 

Note that any existing goto commands will remain unchanged, so remember to verify they are still pointing to the right lines after doing the insert.

 

Type /cmd help add§ to learn about the ascon and asop options.

 

EXAMPLES:

/cmd insert hello 5 /spawn

register

---------

 

Use this command to register a custom command to be recognized as a proper /command. When registered, your command will be recognized by the game, as well as by the /help command.

 

/cmd register <command_name>

 

If you register a custom command with the same name as an already existing regular command, your procedure will be used to override the standard functionality. You can still call the original command from within the procedure.

 

Note that while newly registered commands can be used even from within the console, overriding existing commands only works in the player context.

 

You can control whether newly created commands are automatically registered by modifying the register-new-commands config option.

 

----------

unregister

-----------

 

Use this command to stop your command procedure from being automatically registered as a proper command when starting the server. Since PocketMine does not currently support unregistering a single command, you have to restart your server for the command to finally stop working.

 

/cmd unregister <command_name>

 

You can control whether newly created commands are automatically registered by modifying the register-new-commands config option.

trigger

--------

 

MadCommands is able to trigger your custom command procedures when specific things happen in the game.

 

To make your command procedure automatically run whenever the desired event happens, use the following command:

 

/cmd trigger <event_name> [to|from <area>] <command_name>

 

To unassign your command procedure from the event, simply use the same command without specifying command name:

 

/cmd trigger <event_name> [to|from <area>]

 

For the move event, area can be specified either as a named area defined by LeetProtector or as a combination of level name and coordinates in one of the following forms:

 

<level> <x1> <z1> <x2> <z2>

<level> <x1> <y1> <z1> <x2> <y2> <z2>

<level>[<x1>,<y1>,<z1>|<x2>,<y2>,<z2>]

 

To list all current triggers, simply type /cmd trigger

 

When an event triggers your command procedure it passes some event-specific data to it as parameters. To learn more about individual events supported by this version of MadCommands, type §/cmd help events.

 

EXAMPLES:

/cmd trigger

/cmd trigger login welcome

/cmd trigger death

/cmd trigger move from world 0 0 255 255 leftspawn

onerror

--------

 

Set specific command line to go to in case of error:

You can use this command to create your own error handler to avoid runtime errors from appearing to the users. You can check the %last_error_message% variable to find out what the error is and %last_error_line% for finding out on which command line it has occurred.

 

EXAMPLE:

1 onerror goto 20

2 let %x% = %y% / 0

3 exit

sleep

------

 

Use the sleep§ command to wait for certain amount of time before performind the following commands:

 

sleep <seconds>

 

Note that when used in the main thread, this command schedules a delayed task to execute the remaining commands. When the scheduled task is started, the command procedure timeout is reset as if the procedure was just starting, so you can effectively use the sleep command to keep a long procedure running without timing out and without blocking the server for too long.

 

Using sleep in async mode causes the command procedure to simply wait, and timeout defined in the plugin configuration still applies.

 

Check the proc-max-time and proc-max-time-async config options to find out what timeouts are applies in sync and async execution modes respectively.

 

EXAMPLE:

1 sleep 600

2 message %p% Ten minues elapsed, the eggs must be finished now

schedule

---------

 

Use this command to schedule another command procedure to be executed later:

 

schedule <seconds> <command_name> [<parameter>] ...

 

This command works almost like /cmd run except for that it does not run the procedure immediately but instead schedules it to run later.

 

The specified command procedure needs to be enabled but does not have to be registered as command.

 

You cannot schedule any commands using this command, only command procedures created by MadCommands. You can, however, schedule the same procedure, turning it to a regularly repeated task.

 

EXAMPLE:

1 /say ding-dong

2 schedule 3600 chime

 

-----

async

------

 

This command switches the currently running command procedure to the asynchronous mode.

 

async

 

In asynchronous mode, the following commands are executed in an asynchronous task, not blocking the main server thread.

 

Use this to perform actions that take long time, such as complex calculations, and do not need to interact with the game, to avoid unnecessary lags.

 

Note that game commands cannot be used in asynchronous mode because accessing the main thread's data structures from an async task would cause the server to crash.

 

Use the sync command to switch back to synchronous execution mode.

 

EXAMPLE:

1 let %inventory% = playerinventory(%p%)

2 async

3-19 examine and adjust player's inventory

20 sync

21 carry %p% %inventory§

if

---

 

The if-then command lets you execute command only if certain condition is met. The syntax loks like this:

 

if <expression> then <command> [<parameter>] ...

 

What it does is that it first evaluates the expression entered between the word if and the word then. If the result of the expression is true or a valid non-zero number, or a non-empty string, then the command entered after the word then is executed. If the result of the expression is false or zero, or an empty string, then the rest of the command is simply ignored.

 

This is the most important command to be able to implement any kind of logic and decision-making in your command procedures, so necessary to make complex features, such as minigames.

 

EXAMPLES:

1 ascon if %p% = "madhon" then /op %p%

2 if %y% > 127 then message %p% You're flying too high

3 load %past_participants%

3 if %p% notin %past_participants% then goto 10

4 message %p% You have already played this game

5 exit

let

----

 

Use the let command to assign value to a variable.

 

MadCommands requires every variable name to be enclosed by the percent (%) signs, i.e. %x% or %mylongvariablename%. This limitation is to make the plugin slightly easier to develop and faster to parse your commands. It makes it also easy to use variables in game commands and have them replaced with their values when executing the commands.

 

To simply assign result of an expression to a variable, use:

 

let %<variable_name>% = <expression>

 

MadCommands also supports arrays (one variable containing multiple values), both sequential and associative. If you want to add a new value to the end of an array, use empty square brackets:

 

let %<variable_name>%[] = <expression>

 

To assign value to a specific index of an array, include the desired index in the square brackets. Index can be either number or string:

 

let %<variable_name>%[<number>] = <expression>

let %<variable_name>%["<text>"] = <expression>

 

You can also use another variable as an index. In that case the value of the variable is used as index:

 

let %<variable_name>%[%<variable_name>%] = <expression>

 

To delete variable, simply assign nothing to it:

 

let %<variable_name% =

let %<variable_name>%[<number>] =

let %<variable_name>%["<text>"] =

 

You can use any combination of letters, numbers, underscores, and dashes as variable name, as long as it starts with a letter.

 

You cannot, however, use the let command to assign values to any of the pre-defined variables, such as %p%, %x%, or %time%. Type /cmd help variables to learn about pre-defined variables.

 

You can use any variable as part of any command's parameters. Each variable will be replaced with its value when executing the command. You cannot, however, directly reference array elements this way. To use an array element in a command, you first have to assign it to a dedicated variable.

 

The let command can be implicit. You can simply omit it and just begin the command with the variable name.

 

EXAMPLES:

1 let %a% = -123

2 let %b% = "bbb"

3 let %c% = %a%

4 let %d%[3] = "thre

5 let %d%[5] = "five

6 let %d%[] = "six"

7 let %e%["a"] = %a%

8 let %e%["d"] = %d%

9 %e%[%b%] = %b%

save

-----

 

Variables normally exist only as long as your procedure is running. To keep value of any variable for later user, you can use the save command:

 

save %<variable_name>% [<version>]

 

Saved variable survives even server restarts, so is very useful for storing things like minigame leaderboards or per-player settings. Use the load command to reload previously saved variable.

 

You can specify optional <version> to save several versions of the same variable. This parameter can be any valid expression, so it can be calculated. This can be very handy, for instance, to store different value for each player, without having to keep values for all players in one huge array variable.

 

Only letter, numbers, dashes, and underscores can be used in <version>. All other characters are automatically stripped. Letters are case insensitive. I.e. "Test" and "t*E*s*T" will be considered the same version.

 

You can use the save command from the async execution mode but be careful not to try to save the same variable from two places at the same time. It is generally safer to only save variables in the synchronous mode.

 

EXAMPLE:

1 load %members%

2 if %p% notin %members% then let %members%[] = %p%

3 save %members%

4 let %playerdata% = playerstatus(%p%)

5 save %playerdata% %p%

 

----

load

-----

 

Use the load command to reload any variable previously saved using the save command:

 

load %<variable_name>% [<version>]

 

If you have used the optional <version> parameter when saving the variable, you have to provide the exact same version specification in order to be able to load it.

 

See help for the save command for more details.

 

EXAMPLE:

1 load %members%

2 if %p% notin %members% then let %members%[] = %p%

3 save %members%

message

--------

 

The message command displays message to a player or list of players. The message appears in the chat just like any other message.

 

To display message to a single player, provide the player's name as first parameter and the message as second parameter:

 

message <player_name> <message>

 

You can provide a valid expression as a message. Otwherwise everything after the second space is considered the message.

 

To send a message to multiple players, provide a valid expression resulting in an array as the first parameter:

 

message <players_array> <message>

 

The message will then get broadcasted to all players listed in the array.

 

EXAMPLES:

1 message %p% This is simple message

2 message %p% "This is expression message"

3 message %p% %args%[1+]

4 message onlineplayers() This is simple broadcast

5 message onlineplayers() "This is\nmulti-line\nbroadcast"

drop

-----

 

Use the drop command to drop an item at certain position of the game world/map, just like if it was dropped by a player or destroyed chest. You need to specify the level name and coordinates of the position where the dropped item is supposed to spawn:

 

drop <level> <x> <y> <z> <item_id>[:<meta>]

 

Any player will then be able to collect the item, until it is automatically despawned.

 

EXAMPLES:

1 drop world 81 71 47 diamond_axe

2 let %items% = array("diamond_sword", "gold_sword", "stone_sword", "wooden_sword")

3 let %rnd% = random(0, 3)

4 let %randomitem% = %items%[%rnd%]

5 drop world 81 81 47 %randomitem%

block

------

 

You can use the block command to change a single block in the game world/map to anything you want. You need to specify the level name and coordinates of the block you want to change, and then the block id and optional metadata:

 

block <level> <x> <y> <z> <block_id>[:<meta>]

 

If you're setting a two-block item, such as bed ir double-chest (currently the only kinds of blocks supported this way), you have to specify coordinates of the second block as well:

 

block <level> <x> <y> <z> <block_id>[:<meta>] <x2> <y2> <z2>

You can specify the block by its numeric ID (i.e. 1 for stone), or by its name defined by PocketMine (i.e. "stone").

 

WARNING: Although you can set multiple blocks in a loop, this command is not intended for automated building or filling of large areas, because every single block change produces update to be sent to all players in visible distance just like if a player in creative mode placed the block there. Setting too many blocks by a single procedure might cause significant lags. However, you can efficiently use this for resetting minigame elements, opening or closing hidden doors, etc.

 

EXAMPLES:

1 block world 81 71 47 stone

2 block world 82 71 47 1:1

3 block world 83 71 47 bed:3

4 block world 84 71 47 26:11

5 block world 86 71 47 bed 85 71 47

store

------

 

Use the store command to store items in a block's inventory:

 

store <level> <x> <y> <z> <items_array>

 

Currently only chests, trapped chests, and item frames are supported.

 

For item frames, the supplied items array is supposed to contain only one item. If you specify the item as an array also containing the "ItemRotation" node, you can have the item in the item frame rotated as desired.

 

The easiest way to prepare an array of items for use with the store command is to get it from another block of the same kind using the blockcontents function.

2 block world 81 71 47 chest

------

 

Produce game sound at specified position, hearable either by all nearby players, or by specified players:

 

sound <level> <x> <y> <z> <sound_id> [<modifier> [<entity_type> [<players>]]]

 

The sound will be played like if it was caused at the specified place.

 

<sound_id> can be either numeric sound ID or sound name. Valid numeric IDs are between 0 and 254 and between 1000 and 1063.

 

Some sounds can have different variants or volume levels. You can select variant by providing numeric <modifier>. If you do not provide it, default sound will be produced. For note sound the modifier specifies tone. For some sounds, such as levelup very high values (over 100000000) need to be provided to make the sound hearable.

 

Some useful sounds:

ambient, ambient_baby, ambient_in_water, death, hurt, eat, explode, fire, ignite, portal, portal_travel, levelup, power_on, power_off, note, armor_equip_elytra, glass, potion_brewed, camera_take_picture, door

 

Some sounds, such as ambient or death need <entity_type> to be specified. You can provide either numeric entity ID, or entity name, such as player, creeper, or zombie. Valid entity IDs are, at the time of writing this, from 10 to 113.

 

Some useful entities:

player, creeper, zombie, spider, villager, enderman, shulker, ghast, ender_dragon

If you provide also name of one or more <players>, the sound will be heared only by those players. Otherwise all players close enough to the specified position will hear it.

2 sound world 84 71 45 ambient 0 zombie

3 sound world 84 71 45 1003

4 sound world 84 71 45 levelup 100663296 madhon

Operators

----------

 

You can use the following binary operators in complex expressions:

 

= - results in true if both sides are equal

!= - results in true if each side is different

> - results in true if left side is greater than right side

>= - results in true if left side is greater or equal to right side

< - results in true if left side is lower than right side

<= - results in true if left side is lower or equal to right side

+ - add two numbers/strings/arrays

- - subtract second number/string/array from first one

/ - divide first number by second number

* - multiply two numbers

& - apply bitwise AND between two numbers

| - apply bitwise OR between two numbers

^ - apply bitwise XOR between two numbers

and - true if both sides are true

or - true if at least one of the sides is true

in - true if value on the left exists in array/string on the right

notin - true if value on the left doesn't exist in array/string on the right

 

If you search for array in string using the in and notin operators, left operand is considered found in the right operand as long as at least one of the array elements exists in the string.

substring

----------

In this function <start> is zero-based index of the first character to extract from the string. So if you pass 0 there, you will be extracting beginning of the string. You can also think of it as number of characters to skip from the beginning of the string.

substring("Hello World, it's me, madhon", -6, 6)

dialog

-------

 

dialog(<player>, <form_data>) shows dialog to player and returns response

 

This function can be used to prompt specific player for some information and get it back as a response.

 

<form_data> must be an associative array, containing the following elements:

 

["title"] text string to use as a dialog title

["type"] must be one of: "modal" | "form" | "custom_form"

 

Depending on the dialog type, additional values must be provided.

 

For "modal" dialog (simple dialog with two buttons):

 

["content"] text string to display in the dialog as description

["button1"] name of first button, pressing this will return true

["button2"] name of second button, pressing this will return false

 

Result of the "modal" dialog is a boolean value.

 

For "form" dialog (simple form with one ore more buttons and optional images):

 

["content"] text string to display in the dialog as description

["buttons"] associative array of button definitions:

["buttons"]["text"] button name

["buttons"]["image"] array containing image parameters

["buttons"]["image"]["type"] image type (only "url" supported)

["buttons"]["image"]["data"] image url

 

Result of the "form" dialog is either zero-based index of the pressed button, or boolean false if the player has closed the dialog without pressing a button.

 

For "custom_form" dialog:

 

["content"] array of dialog items, each item represented by an array with the following elements:

 

["content"]["type"] one of "label" | "input" | "dropdown" | "slider" | "step_slider" | "toggle"

 

["content"]["text"] text to display as label

["content"]["default"] default string value for "input" item

["content"]["placeholder"] value placeholder for "input" item

["content"]["options"] array of values for "dropdown" item

["content"]["min"] minimum value for "slider" item

["content"]["max"] maximum value for "slider" item

["content"]["steps"] array of step names for "step_slider" item

["content"]["default"] either %true% or %false% for "toggle" item

 

Result of the "custom_form" dialog is an array of values entered in individual items, or boolean false if the player has closed the dialog without pressing the Submit button.

 

For "dropdown" and "step_slider" items the value returned is zero-based index of the selected option.

 

Command procedure execution is paused while waiting for the player's response. In the case of the player leaving server without responding to the dialog, execution is resumed and the function returns boolean false.

Events

-------

 

The following events can be set to trigger custom MadCommands procedure.

Procedure does not have to be registered as command.

When event triggers your procedure it will pass key information about the event to the

procedure as parameters.

Some events can be cancelled using the cancel command if it is used soon enough.

 

startup triggers when the server finishes starting up

shutdown triggers when the server is shutting down (or the plugin is disabled)

login triggers when player joins the server

leave triggers when player leaves the server

spawn triggers when player is spawned or respawned

bed triggers when player enters or leaves a bed (entering bed can be cancelled)

gm triggers when player switches gamemode (can be cancelled)

eat triggers when player eats food (can be cancelled)

chat triggers when player writes message in chat (can be cancelled)

damage triggers when player or mob gets damage (can be cancelled)

death triggers when player dies

cheat triggers when player tries to cheat (can be cancelled)

place triggers when player places a block (can be cancelled)

break triggers when player breaks a block (can be cancelled)

touch triggers when player touches a block (can be cancelled)

appear triggers when a new entity appears

disappear triggers when an entity disappears

hit triggers when projectile hits a block or an entity

move triggers when player moves to or from specified area

 

Type /cmd help <event> to get detailed help for a specific event.
